# Device Tracker V2 - نظام تتبع الأجهزة المتقدم

نظام ويب متقدم لتتبع وإدارة الأجهزة المتصلة بالشبكة مع ميزات متقدمة للمراقبة والإدارة والتقارير.

## المميزات الرئيسية ✨

### 1. إدارة الأجهزة (Device Management)
- إنشاء وتحرير وحذف الأجهزة
- عرض قائمة الأجهزة مع البحث والفلترة
- تتبع حالة الجهاز (متصل/غير متصل/صيانة/معطل)
- معلومات تفصيلية لكل جهاز (IP، MAC، الموقع، البرنامج الثابت)
- صفحة تفاصيل شاملة لكل جهاز

### 2. نظام الصلاحيات (RBAC)
- ثلاث مستويات من الأدوار: مسؤول (Admin)، مدير (Manager)، مستخدم (User)
- إدارة الصلاحيات على مستوى الجهاز
- منح وسحب الصلاحيات
- تتبع التغييرات في الصلاحيات

### 3. لوحة التحكم (Dashboard)
- إحصائيات فورية عن الأجهزة
- رسوم بيانية تفاعلية (Recharts)
- عرض حالة الأجهزة
- روابط سريعة للعمليات الشائعة
- حالة النظام والاتصال

### 4. سجل الأنشطة (Activity Logs)
- تسجيل جميع العمليات على الأجهزة
- البحث والفلترة حسب نوع الإجراء
- تصدير CSV
- معلومات المستخدم والتاريخ والوقت

### 5. التنبيهات والإشعارات (Alerts & Notifications)
- إنشاء تنبيهات تلقائية عند قطع الاتصال
- مستويات خطورة مختلفة (منخفض/متوسط/عالي/حرج)
- حل التنبيهات يدويًا
- فلترة التنبيهات النشطة والمحلولة

### 6. الخريطة التفاعلية (Interactive Map)
- عرض مواقع الأجهزة
- قائمة الأجهزة مع الفلترة
- روابط إلى تفاصيل الجهاز
- إحصائيات الأجهزة بالموقع

### 7. التقارير (Reports)
- تقارير الأجهزة مع الإحصائيات
- تقارير الأنشطة
- تقارير التنبيهات
- تصدير CSV
- فلترة حسب التاريخ

### 8. إدارة المستخدمين (User Management)
- عرض قائمة المستخدمين
- تغيير أدوار المستخدمين
- البحث والفلترة
- معلومات المستخدم (القسم، الهاتف، آخر تسجيل دخول)

## المتطلبات التقنية 🛠️

### المتطلبات الأساسية
- Node.js 18+ و npm/pnpm
- MySQL 8.0+
- متصفح حديث يدعم ES2020+

### المكتبات الرئيسية
- **Frontend**: React 19, TypeScript, Tailwind CSS, Recharts
- **Backend**: tRPC, Drizzle ORM, Express
- **Database**: MySQL
- **UI Components**: Shadcn UI
- **Routing**: Wouter

## التثبيت والتشغيل 🚀

### 1. استنساخ المستودع
```bash
gh repo clone kingebrahimopq-create/device-tracker-v2
cd device-tracker-v2
```

### 2. تثبيت المتطلبات
```bash
pnpm install
```

### 3. إعداد متغيرات البيئة
```bash
# قم بإنشاء ملف .env.local وأضف المتغيرات المطلوبة
cp .env.example .env.local
```

### 4. إعداد قاعدة البيانات
```bash
# تشغيل الهجرات
pnpm run db:migrate
```

### 5. تشغيل المشروع
```bash
# تشغيل الخادم والعميل معًا
pnpm run dev
```

سيكون التطبيق متاحًا على `http://localhost:5173`

## نظام الصلاحيات 🔐

### الأدوار المتاحة

**مسؤول (Admin)**
- الوصول الكامل لجميع الميزات
- إدارة المستخدمين وتعيين الأدوار
- عرض جميع الأجهزة والسجلات
- إدارة الصلاحيات

**مدير (Manager)**
- إدارة الأجهزة (CRUD)
- عرض جميع السجلات والتنبيهات
- إنشاء التقارير
- إدارة صلاحيات الأجهزة

**مستخدم (User)**
- عرض الأجهزة المسموح بها
- عرض السجلات الخاصة به
- عرض التنبيهات

## قاعدة البيانات 🗄️

### الجداول الرئيسية

**users** - جدول المستخدمين
- معرف المستخدم
- البريد الإلكتروني والاسم
- الدور والقسم
- معلومات الاتصال

**devices** - جدول الأجهزة
- معرف الجهاز الفريد
- الاسم والوصف
- الحالة والموقع
- معلومات الشبكة (IP، MAC)
- البيانات الوصفية

**device_permissions** - جدول الصلاحيات
- المستخدم والجهاز
- مستوى الصلاحية (view/edit/admin)
- تاريخ منح الصلاحية

**activity_logs** - جدول السجلات
- المستخدم والجهاز
- نوع الإجراء والوصف
- البيانات القديمة والجديدة

**alerts** - جدول التنبيهات
- الجهاز والنوع
- مستوى الخطورة والرسالة
- حالة التنبيه

**notifications** - جدول الإشعارات
- المستخدم والتنبيه
- النوع والحالة
- طريقة الإرسال
